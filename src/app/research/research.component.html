  • 3D and Parsing Project at nVent Thermal Pvt Ltd

    Objective: Develop a proof-of-concept (POC) for rendering 3D piping systems by parsing PCF/IDF files and mapping data into JSON format for frontend visualization using Three.js.

    Project Overview:

    The *3D Piping System POC* project focused on creating a system to visualize piping systems in 3D. The project involved the following key components:

    • File Parsing: Developed a .NET microservice to parse PCF (Piping Component File) and IDF (Isogen Data File) formats and convert them into JSON data.
    • Frontend Rendering: Utilized Three.js to render the parsed 3D data in a web application, providing an interactive and dynamic visualization of piping systems.
    • Integration: Integrated the JSON data with the frontend application to visualize complex piping systems accurately.
    • Proof-of-Concept Development: Created a functional prototype to demonstrate the feasibility and effectiveness of the 3D rendering approach.

    Technologies Used:

    • Three.js: JavaScript library for creating and displaying animated 3D graphics in a web browser.
    • .NET: Framework for developing the microservice responsible for parsing and converting PCF/IDF files.
    • JSON: Data format used for transferring parsed data between the backend and frontend.

    Key Achievements:

    • Successfully developed a microservice to parse and convert piping system data into a usable format.
    • Implemented 3D visualization of complex piping systems using Three.js, enhancing the user's ability to interact with and understand the data.
    • Demonstrated the feasibility of the approach through a functional proof-of-concept, showcasing its potential for further development and application.
    • Integrated backend and frontend components to create a cohesive and interactive visualization tool.

  • Flajolet-Martin Algorithm Research Project

    Objective: Evaluate and enhance distinct pixel value counting in video streams using the Flajolet-Martin (FM) algorithm.

    Approaches Compared:

    • Average Approach: Used multiple hash functions; averaged the results.
    • Median Approach: Applied multiple hash functions; used the median of average results.
    • Combined Approach: Grouped maximum values from hash functions, averaged within groups, and used the median of these averages.

    Conclusion:

    • FM Algorithm Efficiency: The standard FM algorithm with one hash function performed efficiently and provided good results.
    • Median and Combined Approaches: Both the median and combined approaches (averaging or median of all hash results) showed superior performance, maintaining accuracy even with increased discrepancies between actual and estimated distinct counts.
    • Average Approach Limitations: The average of all hash results performed poorly.
    • Recommendation: For quick results, use the standard FM algorithm. For optimal accuracy, especially with high computational resources, the median or combined approach is preferable.
